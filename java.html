Définition et utilisation dans le développement logiciel

Java est un langage de programmation orienté objet qui offre une syntaxe claire et concise, ainsi qu'une vaste bibliothèque standard. Sa popularité est le résultat de sa portabilité, de sa robustesse et de sa facilité d'utilisation. En tant qu'experte en ingénierie du logiciel, la maîtrise de Java me permet d'écrire du code clair, maintenable et extensible, ce qui est essentiel pour créer des applications de qualité.

La compétence technique en Java est d'une importance cruciale pour un développeur, car elle ouvre de nombreuses opportunités professionnelles. Java est utilisé dans une grande variété de domaines, tels que le développement d'applications web, les systèmes embarqués, les applications de bureau, les applications mobiles et même l'intelligence artificielle. Avoir une bonne maîtrise de Java permet de travailler sur divers projets et de s'adapter à différents contextes.

L'un des avantages majeurs de ce langage est sa portabilité. Le code source Java peut être compilé en bytecode, qui est exécuté par la machine virtuelle Java (JVM). Cela signifie que les applications Java peuvent s'exécuter sur différents systèmes d'exploitation, tels que Windows, macOS et Linux, sans nécessiter de modifications supplémentaires. Pour un développeur, cela ouvre de vastes possibilités et permet de cibler un large éventail de plateformes, offrant ainsi une plus grande accessibilité aux utilisateurs finaux.

De plus, Java possède une bibliothèque standard complète qui offre une multitude de fonctionnalités prêtes à l'emploi. Cette bibliothèque comprend des classes et des méthodes pour gérer les opérations courantes, telles que la manipulation de chaînes de caractères, la gestion des entrées/sorties, la gestion des fichiers, le traitement des erreurs, etc. En tant qu'experte en ingénierie du logiciel, cela me permet de gagner du temps en utilisant ces composants prêts à l'emploi, plutôt que de devoir les implémenter moi-même.

Mon expérience autour de cette technologie


Au cours de ma carrière, j'ai pu appliquer mes compétences en Java de manière significative.

En effet, lors du projet FBA, nous étions avec mon groupe chargés d'organiser un événement au cours duquel les étudiants de semestre 1 de mon école, Intech, devaient présenter à un jury des bandes-annonces qu'ils avaient montées. Afin de simplifier le système de notation, nous avons décidé en tant qu'organisateurs de créer une application web sur laquelle les membres du jury pouvaient se connecter afin de noter individuellement les créations. Cela nous permettait de disposer d'un moyen fiable calculant automatiquement les résultats et réduisant considérablement les erreurs de calcul.

Le défi était de mettre en place une interface conviviale, de gérer les interactions en temps réel et de garantir la sécurité des données. De plus, le projet avait des contraintes de délai strictes, nécessitant une livraison rapide et efficace. Grâce à ma connaissance approfondie de Java et à mon expérience avec Spring Boot, j'ai pu relever ce défi avec succès. J'ai utilisé Spring Boot pour faciliter le développement de l'application, en profitant de son architecture légère et de sa configuration par défaut. Cela m'a permis de me concentrer sur la logique métier de l'application plutôt que de passer du temps à configurer l'infrastructure. De plus, Spring Boot m'a fourni une intégration transparente avec d'autres technologies clés, telles que Spring MVC pour la gestion des requêtes HTTP. Grâce à ces fonctionnalités, j'ai pu développer rapidement une interface utilisateur réactive et intuitive, permettant au jury de noter les bandes-annonces de manière fluide et sans heurts.

Une autre force de Java dans ce projet était sa gestion efficace de la sécurité. Nous avons en effet pu mettre en place des mécanismes d'authentification et d'autorisation solides pour protéger les données sensibles et garantir que seuls les membres autorisés du jury pouvaient accéder à l'application. Cela a grandement contribué à la confiance des utilisateurs et à la crédibilité de l'application.

Cette anecdote met en évidence comment ma compétence technique en Java et l'utilisation de Spring Boot ont permis de relever un défi complexe dans un délai serré. La combinaison de la polyvalence de Java, de la simplicité de développement offerte par Spring Boot et de l'écosystème solide de bibliothèques et de frameworks a été essentielle pour atteindre avec succès les objectifs du projet.

Outre ce projet, j'ai également pu appréhender le langage Java à travers le développement de la solution mobile du projet e-Rescue, un projet d'application Android au profit du Comité Côte-Basque des Maîtres-Nageurs Sauveteurs, une association qui forme ses adhérents au sauvetage en mer. Ce projet m'a permis de découvrir une facette différente de Java, à travers des différences techniques.

En effet, les applications Android sont exécutées sur des appareils mobiles, ce qui signifie qu'elles doivent prendre en compte les ressources limitées, telles que la mémoire, la batterie et la puissance de calcul. Le développement Android nécessite une compréhension approfondie du cycle de vie des activités, de la gestion des fragments, des services en arrière-plan, etc. Le développement d'une application Android sous Java et le développement backend en Spring Boot partagent certains concepts, ils diffèrent cependant en termes de plateforme de développement, d'environnement d'exécution, d'architecture, de bibliothèques et de frameworks utilisés, ainsi que du domaine d'application final. Chacun de ces domaines de développement a ses propres considérations et bonnes pratiques spécifiques et cela a été très formateur pour moi d'expérimenter ces différences.

Et demain ?

Java est un langage qui évolue très rapidement, nous en sommes actuellement à la version 19. De nombreuses fonctionnalités sont régulièrement ajoutées, il me reste donc encore beaucoup à apprendre. Je n'ai à l'heure actuelle pas encore eu l'opportunité d'utiliser Java en entreprise. Il s'agit cependant tout de même d'un langage vis-à-vis duquel je me sens à l'aise et qui me permet d'être présente sur ce marché très demandé.

Avec le recul que j'ai aujourd'hui je ne peux évidemment que conseiller l'apprentissage de Java. En position de développeuse n'ayant pas utilisé ce langage depuis près de deux ans, je sais cependant qu'il ne serait pas bien compliqué de me remettre en selle, ce langage ne comportant pas, selon moi, de complexité particulière et étant similaire à la plupart des autres langages de programmation orientés objet. Ma formation à ce langage a été réalisée de manière suffisamment complète durant ma formation, à un tel point que je ne ressens pas aujourd'hui le besoin de me former davantage.

package com.intech.erescue;
import android.content.Intent;
import android.os.Bundle;
import android.support.annotation.NonNull;
import android.support.v4.app.FragmentManager;
import android.support.v4.app.FragmentTransaction;
import android.support.v7.app.AppCompatActivity;
import android.support.v7.widget.AppCompatButton;
import android.view.View;
import android.widget.LinearLayout;
import android.widget.RelativeLayout;
import android.widget.Toast;

import com.google.gson.JsonObject;
import com.intech.R;
import com.intech.erescue.api.APICaller;
import com.intech.erescue.api.APIExtractor;
import com.intech.erescue.calendar.decorators.EventDecorator;
import com.intech.erescue.calendar.decorators.HighlightWeekendsDecorator;
import com.intech.erescue.calendar.decorators.MySelectorDecorator;
import com.intech.erescue.calendar.decorators.OneDayDecorator;
import com.intech.erescue.model.Event;
import com.intech.erescue.utility.SessionUtils;
import com.prolificinteractive.materialcalendarview.CalendarDay;
import com.prolificinteractive.materialcalendarview.MaterialCalendarView;
import com.prolificinteractive.materialcalendarview.OnDateSelectedListener;

import org.threeten.bp.DayOfWeek;
import org.threeten.bp.LocalDate;
import org.threeten.bp.Month;

import java.util.ArrayList;
import java.util.Collections;
import java.util.List;


public class TrainingCalendar extends AppCompatActivity implements OnDateSelectedListener {
    public static final int DIALOG_QUEST_CODE = 300; //L'ID pour la requête d'un DialogBox
    private final OneDayDecorator oneDayDecorator = new OneDayDecorator(); //La décoration principale pour les jours sélectionnés (grossissement + mise en gras)
    MaterialCalendarView widget; //Le calendrier
    List<<Event> events = new ArrayList<>(); //La liste des événements
    private AppCompatButton btn_legend;
    private LinearLayout legend;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.a_activity_calendar_training);
        widget = findViewById(R.id.calendarTraining);
        btn_legend = findViewById(R.id.btn_legend);
        legend = findViewById(R.id.legend);

        legend.setVisibility(View.GONE);

        checkRank();
        showLoader();

        //Initialisation du calendrier
        widget.setOnDateChangedListener(this);
        widget.setShowOtherDates(MaterialCalendarView.SHOW_ALL);

        //On recupère la date d'aujourd'hui
        final LocalDate instance = LocalDate.now();
        widget.setSelectedDate(instance);

        //Paramétrage des dates minimum et maximum (1 er janvier de l'année précédente au 31 décembre de l'année suivante)
        final LocalDate min = LocalDate.of(instance.getYear()-1, Month.JANUARY, 1);
        final LocalDate max = LocalDate.of(instance.getYear()+1, Month.DECEMBER, 31);

        //On applique les dates min/max et on paramètre le premier jour de la semaine (ici lundi mais sinon dimanche par défaut)
        widget.state().edit().setFirstDayOfWeek(DayOfWeek.MONDAY).setMinimumDate(min).setMaximumDate(max).commit();

        //On ajoute les décorations
        widget.addDecorators(
                //Décoration des jours sélectionnables
                new MySelectorDecorator(this),
                //Décoration du jour sélectionné
                oneDayDecorator,
                //Décoration du background des weekends
                new HighlightWeekendsDecorator()
        );


        //Ajout des événements
        APICaller callApi = new APICaller(new APICaller.AsyncResponse() {
            @Override
            public void processFinish(JsonObject callbackResponse) {
                //Action à effectuer avec les données récupérées (callbackResponse)
                events.clear();
                hideLoader();
                for (int i = 0; i<APIExtractor.jsonLen(callbackResponse); i++){
                    events.add(new Event(callbackResponse, i));
                }
                for (Event event : events) {
                    widget.addDecorator(new EventDecorator(event.getLevelEvent(), Collections.singleton(event.getCalendarDayEvent())));
                }
            }
        });
        callApi.execute("GET_EVENT_FOR_USER", "iduser", SessionUtils.getFromSession("uid", this));

        //Initialisation des ajouts d'evénement
        ((AppCompatButton) findViewById(R.id.btn_add_event)).setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                showDialogFullscreen();
            }
        });

    }

    /**
     * Méthode qui permet de chercher le rôle de l'utilisateur actuel dans la base de données distante
     * Cache le bouton "Ajouter un événement" s'il s'agit d'un adhérent
     */
    public void checkRank() {
        APICaller callApi2 = new APICaller(new APICaller.AsyncResponse() {
            @Override
            public void processFinish(JsonObject callbackResponse) {
                try {
                    callbackResponse.get("data");
                    System.out.println(APIExtractor.jsonExtractor(callbackResponse, 0, "nomrang"));
                    if (APIExtractor.jsonExtractor(callbackResponse, 0, "nomrang").equals("ADHERENT")) {
                        disableButton();
                    }
                } catch (Exception e) {
                    System.err.println(e);
                }
            }
        });
        callApi2.execute("GET_USER_BY_ID", "iduser", SessionUtils.getFromSession("uid", this));
    }

    public void disableButton() {
        AppCompatButton addEvent = findViewById(R.id.btn_add_event);
        addEvent.setVisibility(View.GONE);
    }

    @Override
    public void onDateSelected(
            @NonNull MaterialCalendarView widget,
            @NonNull CalendarDay date,
            boolean selected) {

        oneDayDecorator.setDate(date.getDate());
        widget.invalidateDecorators();

        //Si il y a au moins un événement au jour sélectionné, ouvrir la liste des événéments
        if(isEventThisDay(date)){
            Intent intent = new Intent(getBaseContext(), ListEventOfDay.class);
            intent.putExtra("date", date.getDate().toString());
            startActivity(intent);
        }
    }

    /**
     * Crée un Fragement pour l'ajout d'un nouveau événement (façon dialogBox)
     * Affiche un toast si l'ajout est finalisé : "Événement ajouté"
     */
    private void showDialogFullscreen() {
        FragmentManager fragmentManager = getSupportFragmentManager();
        AddEventForm newFragment = new AddEventForm();
        newFragment.setRequestCode(DIALOG_QUEST_CODE);
        FragmentTransaction transaction = fragmentManager.beginTransaction();
        transaction.setTransition(FragmentTransaction.TRANSIT_FRAGMENT_OPEN);
        transaction.add(android.R.id.content, newFragment).addToBackStack(null).commit();
        newFragment.setOnCallbackResult(new AddEventForm.CallbackResult() {
            @Override
            public void sendResult(int requestCode, Object obj) {
                Toast toast = Toast.makeText(getApplicationContext(), "Événement ajouté", Toast.LENGTH_SHORT);
                toast.show();
            }
        });
    }

    public void showLegend(View view){
        if (legend.getVisibility() == View.VISIBLE) {
            btn_legend.setText("Afficher la légende");
            legend.setVisibility(View.GONE);
        }
        else {
            btn_legend.setText("Masquer la légende");
            legend.setVisibility(View.VISIBLE);
        }
    }

    /**
     * Méthode qui permet d'afficher le panel de chargement
     */
    public void showLoader() {
        RelativeLayout loadingPanel = (RelativeLayout)findViewById(R.id.loadingPanel);
        loadingPanel.setVisibility(View.VISIBLE);
    }

    /**
     * Méthode qui permet de cacher le panel de chargement
     */
    public void hideLoader() {
        RelativeLayout loadingPanel = (RelativeLayout)findViewById(R.id.loadingPanel);
        loadingPanel.setVisibility(View.GONE);
    }

    /**
     * Vérifie si il y a un évènement au jour entré en paramètre
     * @param day le jour cible
     * @return vrai si il en a au moins un, faux sinon
     */
    public  boolean isEventThisDay(CalendarDay day){
        for (Event event : events) {
            if (event.isThisDay(day)){
                return true;
            }
        }
        return false;
    }
}
