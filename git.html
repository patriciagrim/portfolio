Git

Page officielle : https://git-scm.com/


Définition et utilisation dans le développement logiciel

Git est un système de contrôle de version distribué largement utilisé dans le développement logiciel. Il permet de gérer et de suivre les modifications apportées au code source d'un projet. Git enregistre toutes les modifications apportées au code, ce qui facilite la gestion des versions et permet de revenir à des versions précédentes du logiciel. C'est un outil essentiel pour les développeurs, car il offre une manière efficace de gérer l'historique des modifications, de fusionner les différentes branches de développement et de résoudre les conflits qui peuvent survenir lors de la collaboration sur un même projet.

Dans mon métier, Git facilite grandement la collaboration avec d'autres membres de l'équipe de développement. Chaque développeur peut travailler sur sa propre copie du code en utilisant des branches distinctes, ce qui permet de travailler sur différentes fonctionnalités simultanément sans perturber le travail des autres. Git offre également des outils pour résoudre les conflits qui peuvent survenir lorsque plusieurs développeurs modifient le même fichier, en permettant de comparer et de choisir les modifications à conserver.

Git permet également de créer des branches séparées pour développer de nouvelles fonctionnalités. Cette fonctionnalité facilite l'expérimentation et l'itération sur le code sans affecter la branche principale du projet. Une fois les modifications testées et validées, elles peuvent être fusionnées en toute sécurité avec la branche principale.

Enfin, Git s'intègre parfaitement avec des outils d'intégration continue tels que Jenkins. Cette intégration permet d'automatiser les processus de construction, de test et de déploiement, garantissant ainsi un flux de développement continu et cohérent.

Mon expérience autour de cette technologie


Lorsque j'étais en terminale, en 2016, j'ai participé à un projet de création d'un jeu en utilisant Pygame, une bibliothèque Python, avec mon groupe de camarades de classe. Ce jeu servait de projet évalué dans le cadre de la matière "Informatique et Sciences du Numérique", une option du baccalauréat scientifique. L'idée était simple : réviser les notions du baccalauréat de manière ludique et interactive. En tant que joueur, le but était de choisir une matière et de répondre correctement à un maximum de questions. Au nombre de 4 développeurs, nous étions tous passionnés par la programmation et excités par l'idée de développer notre propre jeu. Cependant, notre méthode de collaboration était quelque peu chaotique. À cette époque, nous n'utilisions pas Git, donc après chaque cours, nous devions copier nos versions modifiées du code sur une clé USB. Ensuite, une personne était désignée pour rassembler toutes les modifications et les intégrer manuellement. Cela se traduisait souvent par des problèmes de synchronisation, des fichiers manquants ou des écrasements involontaires de code.

L'anecdote qui me vient à l'esprit est lorsque nous avons passé une journée entière à travailler individuellement sur nos parties du jeu. Nous étions tous très enthousiastes et avons apporté de nombreuses modifications. Cependant, à la fin de la journée, nous avons réalisé que nous avions des versions différentes du code et que nous ne savions pas comment les fusionner. Nous avons dû passer des heures à résoudre manuellement les conflits en comparant les différentes versions, ce qui a entraîné des retards dans notre progression et de la frustration.

Aujourd'hui, avec Git, de tels problèmes seraient évités. L'utilisation de Git facilite grandement la collaboration et la gestion des versions. Chaque membre du groupe pourrait travailler sur sa propre branche, ce qui permettrait de développer les fonctionnalités de manière indépendante sans risquer d'écraser les modifications des autres. Nous pourrions simplement fusionner les branches lorsque nous sommes prêts à intégrer nos modifications, en résolvant automatiquement les conflits grâce aux outils de fusion de Git. De plus, Git permettrait à chaque membre du groupe d'avoir une visibilité instantanée sur les modifications apportées par les autres. Il serait facile de voir qui a fait quelles modifications, d'analyser les différences et de collaborer de manière plus efficace. Les problèmes de synchronisation que nous rencontrions auparavant seraient résolus grâce à Git, qui enregistrerait chaque modification et permettrait à chacun d'accéder à l'historique complet des changements. En utilisant Git, nous aurions également une sauvegarde permanente et sécurisée de notre code. Plus besoin de clés USB ou de transferts manuels. Chaque membre du groupe aurait accès à une copie complète et à jour du projet, ce qui garantirait une meilleure gestion des fichiers et des versions.

Aujourd'hui, grâce à Git, je peux travailler en équipe de manière plus fluide et efficace, en ayant confiance dans la gestion des versions et la collaboration sur mes projets actuels. C'est pour éviter ces problèmes énoncés précédemment que Git est un outil indispensable pour l'ensemble de mes développements. Le portfolio que vous consultez actuellement est d'ailleurs également géré via Git. Cela correspond à 4 années d'expérience autour de cette technologie où j'ai pu mettre en pratique les notions de branches, d'intégration continue via Jenkins et de déploiement afin d'être en capacité de me présenter aujourd'hui comme une experte.

Dans chacun de mes développements qui ont nécessité l'utilisation de Git, cela s'est traduit par l'utilisation d'une plateforme en ligne permettant de visualiser et de gérer mon répertoire Git de manière plus visuelle. J'ai ainsi pu expérimenter plusieurs de ces plateformes, parmi les plus connues.
En effet, alors que tous mes projets étudiants (e-Rescue, Cinétime et FBA) ont été gérés via GitLab, mes projets professionnels (SIGMA et AirMaster-C) sont gérés via Bitbucket, et ce portfolio est géré via GitHub (https://github.com/patriciagrim/portfolio). Cette dernière plateforme est certainement la plus populaire, idéale pour les projets open source et l'intégration continue. De son côté, GitLab met l'accent sur la sécurité et l'auto-hébergement, avec des fonctionnalités avancées disponibles dans sa version libre, tandis que Bitbucket est souvent choisi par les équipes qui utilisent déjà les autres produits d'Atlassian pour bénéficier d'une intégration étroite avec ces outils.

Et demain ?

La maîtrise de Git est selon moi l'une des rares compétences que je considère comme obligatoires lorsque l'on souhaite occuper des fonctions dans le développement de logiciels. J'ai acquis les bases de Git lors des cours donnés sur le sujet pendant ma formation. Cependant, j'estime que mon expertise est aujourd'hui due à l'utilisation quotidienne que j'ai de l'outil.

Bien que j'aie acquis une solide expérience de base avec Git, il existe de nombreuses fonctionnalités avancées qui peuvent améliorer ma productivité et ma compréhension de Git. Je pourrais me plonger dans des sujets tels que les rebases interactifs, les sous-modules, les hooks personnalisés, ou encore les alias de commandes Git. L'exploration de ces fonctionnalités avancées m'aiderait à encore mieux maîtriser Git et à en tirer le meilleur parti dans mes projets.

Pleinement utilisé à travers le monde entier, Git continue d'évoluer et détient devant lui un avenir très prometteur. L'outil vise effectivement à évoluer en se créant des conjonctions avec d'autres outils de développement, tels que des outils de construction ou des plates-formes de déploiement. Autrement dit, les développeurs de Git travaillent aujourd'hui à améliorer l'intégration de Git avec ces outils pour améliorer le flux de travail et c'est en ce sens que je souhaiterais bénéficier de formations sur l'outil afin de continuer à profiter pleinement de ce que Git peut m'offrir.
