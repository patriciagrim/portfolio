Git

Définition et utilisation dans le développement logiciel

Git est un système de contrôle de versions principalement utilisé dans le développement de logiciels. Il permet un suivi des modifications apportées aux projets et de collaborer efficacement avec d'autres développeurs. Pour cela, Git fournit la possibilité d'enregistrer les versions précédentes des projets, de revenir à ces versions si nécessaire, de fusionner des modifications, de tout simplement travailler collaborativement sans craintes.

Via Git, les développeurs travaillent dans leurs propres copies locales du code-source et apportent des modifications et des mises-à-jour dans leur propre branche. Il est ensuite possible de valider les modifications dans le répertoire principal qui contient la version majeure du code-source. D'autres développeurs peuvent ensuite appliquer ces modifications et continuer à travailler sur leurs propres copies locales.

Git est également utilisé pour gérer les conflits de fusion qui surviennent lorsque deux développeurs modifient le même code-source. L'outil permet effectivement de fusionner les modifications de manière transparente, de résoudre les conflits et de maintenir une base de code stable.

Mon expérience autour de cette technologie


Pour ma part, Git fait certainement partie des outils techniques que je maîtrise le mieux. Je n'étais jusqu'à mon entrée en stage pas pleinement consciente de son importance et aujourd'hui, il m'est impossible de développer un quelconque projet sans l'adopter.

Pour vous citer une anecdote quant à son utilité, je pense bien évidemment à l'époque où je ne connaissais pas son existence et précisémment à mon année de Terminale Scientifique. J'avais effectivement pris pour option le module "Informatique et Sciences du Numérique (ISN)", et je devais avec des collègues de classe développer une petite interface de jeu en Python, et c'est ce projet qui permettait de m'évaluer pour le baccalauréat.

S'agissant d'un projet de groupe, nous devions tous collaborer et cela s'est fait en s'échangeant régulièrement nos clés USB comportant nos différents travaux. Une personne disposait de la version finale du projet et était chargée de tout mettre en commun et puis de nous envoyer cette nouvelle version fraîchement à jour. Tout cela était beaucoup trop chronophage et aussi dangereux car le code développé n'était accessible que via nos ordinateurs personnels, il n'était stocké nulle part ailleurs. Je vous laisse ainsi imaginer ma stupéfaction lorsque j'ai découvert Git !

Je me considère aujourd'hui comme experte quant à l'utilisation de Git. J'ai effectivement utilisé cet outil sur plus de 7 projets, représentant plus de 3 ans et demi de pratique, ce site web sur lequel vous êtes actuellement est d'ailleurs également géré via cet outil.

Et demain ?

La maîtrise de Git est selon moi l'une des rares compétences que je considère comme obligatoires lorsque l'on souhaite occuper des fonctions dans le développement de logiciels. J'ai acquis les bases de Git lors des cours donnés sur le sujet pendant ma formation. Cependant, j'estime que mon expertise est aujourd'hui due à l'utilisation quotidienne que j'ai de l'outil.

Pleinement utilisé à travers le monde entier, Git continue d'évoluer et détient devant lui un avenir très prometteur. L'outil vise effectivement à évoluer en se créant des conjonctions avec d'autres outils de développement, tels que des outils de construction ou des plates-formes de déploiement. Autrement dit, les développeurs de Git travaillent aujourd'hui à améliorer l'intégration de Git avec ces outils pour améliorer le flux de travail et c'est en ce sens que je souhaiterais bénéficier de formations sur l'outil afin de continuer à profiter pleinement de ce que Git peut m'offrir.
