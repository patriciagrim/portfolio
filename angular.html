Angular

Définition et utilisation dans le développement logiciel

Développé par Google, Angular est un framework de développement d'applications web basé sur le langage JavaScript. Il offre une structure claire et organisée grâce à l'architecture MVC (Modèle-Vue-Contrôleur), ce qui facilite la maintenance et la collaboration sur les projets de grande envergure. L'un des avantages majeurs d'Angular est sa large gamme de fonctionnalités intégrées, telles que la gestion des routes, la manipulation des formulaires, la validation des données, les animations et la gestion des requêtes HTTP. Ces fonctionnalités prêtes à l'emploi permettent aux développeurs de gagner du temps et d'éviter de réinventer la roue pour les fonctionnalités de base.

L'un des aspects les plus appréciés d'Angular est sa liaison de données bidirectionnelle, qui permet aux modifications des données d'être automatiquement reflétées dans l'interface utilisateur, et vice-versa. Cela offre une expérience utilisateur fluide et réactive. Angular adopte une approche basée sur les composants, ce qui facilite la gestion, la réutilisation et le test des composants individuellement, améliorant ainsi l'efficacité du développement et de la maintenance des applications.

De plus, Angular bénéficie d'une communauté active et d'un vaste écosystème de ressources, ce qui facilite l'accès à une documentation abondante, à des tutoriels et à un support de la communauté. Enfin, l'utilisation de TypeScript, un sur-ensemble de JavaScript, offre une meilleure maintenabilité du code, une détection précoce des erreurs et une expérience de développement améliorée grâce à l'éditeur de code intelligent.

Au 26 janvier 2022, Angular comptait plus de 72 200 étoiles et 1 528 contributeurs sur GitHub.

Mon expérience autour de cette technologie


Dans le cadre du développement de l'application web de la solution e-Rescue, j'ai rencontré un défi majeur. Effectivement, je devais avec mon groupe développer une solution afin de permettre au client, une association de formation au sauvetage maritime, de proposer à ses adhérents et entraîneurs un accès à des fonctionnalités facilitant leur vie associative (gestion des plannings de formation, réservation de matériels, etc.). Pour cela, il a fallu penser à mettre en place une gestion des différents profils d'utilisateurs. Chaque catégorie d'utilisateurs avait des exigences spécifiques en termes de fonctionnalités et d'informations à afficher. Il était essentiel de proposer une interface utilisateur personnalisée pour chaque profil d'utilisateur afin de garantir une expérience optimale. C'est là qu'Angular a été un choix des plus astucieux.

Grâce à ses fonctionnalités de routage et à son architecture modulaire, nous avons pu, avec le groupe de développeurs auquel j'appartenais, créer des composants distincts pour chaque profil utilisateur : adhérent, entraîneur et administrateur. Chaque profil avait son propre ensemble de fonctionnalités spécifiques, mais partageait également certaines fonctionnalités communes. L'utilisation du système de routage d'Angular nous a permis de définir des routes spécifiques pour chaque profil, ce qui a facilité la navigation des utilisateurs vers les fonctionnalités qui leur étaient destinées. Par exemple, les adhérents pouvaient accéder facilement à leur planning de cours, signaler une absence ou commander des articles de merchandising, tandis que les entraîneurs pouvaient gérer les cours de sauvetage et les administrateurs pouvaient gérer les demandes de location et les comptes utilisateurs.

De plus, la structure modulaire d'Angular a grandement facilité la maintenance et l'évolutivité de l'application. Chaque profil utilisateur était développé comme un module indépendant, ce qui signifiait que nous pouvions ajouter ou modifier des fonctionnalités spécifiques à un profil sans perturber les autres. Cela nous a permis de gagner du temps et d'éviter les problèmes de régression lors du développement. En fin de compte, grâce à la flexibilité d'Angular et à sa capacité à gérer efficacement les profils utilisateur, nous avons pu fournir une expérience utilisateur personnalisée et intuitive à chaque catégorie d'adhérents de l'association de sauvetage en mer. Cela a grandement contribué à la satisfaction du client, le Comité Côte-Basque des Maîtres-Nageurs Sauveteurs de Biarritz.

Parallèlement à ce projet, j'ai également participé au développement de la solution Cinétime, une application développée pour l'association Du Cinéma Plein Mon Cartable. Cette association a pour objectif de permettre aux écoliers landais d'accéder à une culture cinématographique en organisant des sorties au cinéma ainsi que des projections itinérantes. La solution visait à rendre ses tâches moins chronophages, en centralisant toutes les données concernant les écoles, les cinémas et les transporteurs.

Plus tard, le projet FBA, pour Festival des Bandes-Annonces, a également hérité d'une solution web développée via cette technologie. Il s'agissait tout simplement d'un événement interne à Intech, l'école auprès de laquelle je me forme, où les étudiants de semestre 1 devaient présenter des bandes-annonces réalisées autour d'un livre lu au préalable. L'application permettait à un jury sélectionné par nos soins de pouvoir noter chaque réalisation rapidement et à nous-même, en qualité d'organisateurs, d'avoir accès aux résultats sans avoir à tout calculer. Enfin, plus récemment, je travaille sur SIGMA, une application web développée pendant mon stage et mon alternance chez Thales Defence Mission Systems, un grand groupe français d'électronique, spécialisé dans la conception de systèmes pour les armées. La solution vise à centraliser les données recueillies pendant les tests d'équipements avioniques embarqués conçus par l'entreprise et de les exploiter.

Aujourd'hui, après toutes ces expériences, il s'agit très certainement de l'un des outils que je maîtrise le mieux dans mon métier, de mon domaine d'expertise. Tous ces projets étudiants étaient à des fins d'assistance par le développement d'outils de gestion et SIGMA, lui, plutôt à des fins de R&D (Recherche & Développement). Cela me confère une certaine aisance en toute situation et celle-ci a certainement été accélérée par ma rapidité de compréhension du fonctionnement d'Angular.

Et demain ?

Aujourd'hui, j'aimerais m'intéresser un peu plus aux outils complémentaires de l'écosystème Angular. Je pense à travers ces mots à RxJS, une bibliothèque puissante pour la programmation réactive. Cela me permettra d'élargir mes compétences et de développer des applications encore plus robustes et scalables.

Aussi, j'aimerais commencer à partager mes connaissances en contribuant à la communauté Angular. En participant à des forums, en répondant à des questions sur Stack Overflow ou en contribuant tout simplement à des projets open source, je pourrais renforcer ma compréhension de cette technologie, apprendre de nouvelles perspectives et pourquoi pas me faire connaître dans la communauté des développeurs Angular.

Puisqu'il s'agit d'une technologie qui est complète et mature, qui a une architecture basée sur les composants, qui possède une large communauté et un large écosystème et qui continue d'évoluer et d'être largement utilisé dans l'industrie, je ne peux aujourd'hui que recommander Angular.

import { AfterViewInit, Component, Input, OnInit, ViewChild } from '@angular/core';

@UntilDestroy()
@Component({
    selector: 'vex-aio-table',
    templateUrl: './gestion-merch.component.html',
    styleUrls: ['./gestion-merch.component.scss'],
    animations: [
        fadeInUp400ms,
        stagger40ms
    ],
    providers: [{
        provide: MAT_FORM_FIELD_DEFAULT_OPTIONS,
        useValue: {
            appearance: 'standard'
        } as MatFormFieldDefaultOptions
    }]
})

export class GestionMerchComponent implements OnInit, AfterViewInit {

    layoutCtrl = new FormControl('boxed');

    /**
    * Simulating a service with HTTP that returns Observables
    * You probably want to remove this and do all requests in a service with HTTP
    */
    subject$: ReplaySubject<Commande[]> = new ReplaySubject<Commande[]>(1);
    data$: Observable<Commande[]> = this.subject$.asObservable();
    customers: Customer[];

    commandes: Commande[];

    @Input()
    columns: TableColumn<Commande>[] = [
        { label: 'Photo Article', property: 'imagemerch', type: 'imagemerch', visible: true },
        { label: 'Numéro de commande', property: 'idcommande', type: 'commande', visible: true },
        { label: 'Nom de l\'objet', property: 'labelmerch', type: 'text', visible: true },
        { label: 'Quantité', property: 'qtcommande', type: 'text', visible: true },
        { label: 'Statut', property: 'namemerchstatus', type: 'statut', visible: true },
        { label: 'Photo Utilisateur', property: 'image', type: 'image', visible: true },
        { label: 'Prénom', property: 'prenomuser', type: 'text', visible: true },
        { label: 'Nom', property: 'nomuser', type: 'text', visible: true },
        { label: 'Contact', property: 'contact', type: 'button', visible: true },
        { label: 'Actions', property: 'actions', type: 'button', visible: true }
    ];
    pageSize = 10;
    pageSizeOptions: number[] = [5, 10, 20, 50];
    dataSource: MatTableDataSource<Commande> | null;
    selection = new SelectionModel<Customer>(true, []);
    searchCtrl = new FormControl();

    labels = aioTableLabels;

    icPhone = icPhone;
    icMail = icMail;
    icMap = icMap;
    icEdit = icEdit;
    icSearch = icSearch;
    icDelete = icDelete;
    icAdd = icAdd;
    icFilterList = icFilterList;
    icMoreHoriz = icMoreHoriz;
    icFolder = icFolder;

    @ViewChild(MatPaginator, { static: true }) paginator: MatPaginator;
    @ViewChild(MatSort, { static: true }) sort: MatSort;

    constructor(private dialog: MatDialog, private API: ApiService) {
    }

    get visibleColumns() {
        return this.columns.filter(column => column.visible).map(column => column.property);
    }

    /**
    * Example on how to get data and pass it to the table - usually you would want a dedicated service with a HTTP request for this
    * We are simulating this request here.
    */
    getData() {
        return of(aioTableData.map(customer => new Customer(customer)));
    }

    ngOnInit() {
        this.API.getAllCommandes().subscribe(res => {
            this.commandes = res.response;
            this.dataSource = new MatTableDataSource();
            this.dataSource.data = this.commandes;
            this.dataSource.paginator = this.paginator;
            this.dataSource.sort = this.sort;
        });

        this.searchCtrl.valueChanges.pipe(untilDestroyed(this)).subscribe(value => this.onFilterChange(value));
    }

    ngAfterViewInit() { }

    updateCustomer(customer: Customer) {
        this.dialog.open(CustomerCreateUpdateComponent, {
            data: customer
        }).afterClosed().subscribe(updatedCustomer => {
            /**
            * Customer is the updated customer (if the user pressed Save - otherwise it's null)
            */
            if (updatedCustomer) {
                /**
                * Here we are updating our local array.
                * You would probably make an HTTP request here.
                */
                const index = this.customers.findIndex((existingCustomer) => existingCustomer.id === updatedCustomer.id);
                this.customers[index] = new Customer(updatedCustomer);
                //this.subject$.next(this.customers);
            }
        });
    }

    onFilterChange(value: string) {
        if (!this.dataSource) {
            return;
        }
        value = value.trim();
        value = value.toLowerCase();
        this.dataSource.filter = value;
    }

    toggleColumnVisibility(column, event) {
        event.stopPropagation();
        event.stopImmediatePropagation();
        column.visible = !column.visible;
    }

    /** Whether the number of selected elements matches the total number of rows. */
    isAllSelected() {
        const numSelected = this.selection.selected.length;
        const numRows = this.dataSource.data.length;
        return numSelected === numRows;
    }

    /** Selects all rows if they are not all selected; otherwise clear selection. */
    masterToggle() {
        //this.isAllSelected() ?
        //this.selection.clear() :
        //this.dataSource.data.forEach(row => this.selection.select(row));
    }

    trackByProperty<T>(index: number, column: TableColumn<T>) {
        return column.property;
    }

    onLabelChange(change: MatSelectChange, row: Customer) {
        const index = this.customers.findIndex(c => c === row);
        this.customers[index].labels = change.value;
        //this.subject$.next(this.customers);
    }

    changeState(commande: Commande, idstatut: number) {
        this.API.changeStateCommande(commande.idcommande.toString(), idstatut.toString()).subscribe(res => {
        const index = this.commandes.findIndex(c => c === commande);
        switch (idstatut) {
            case 1:
                this.commandes[index].namemerchstatus = "Demandé";
                break;
            case 2:
                this.commandes[index].namemerchstatus = "Payé";
                break;
            case 3:
                this.commandes[index].namemerchstatus = "Terminé";
                break;
            default:
                break;
            }
        });
    }

    deleteCommande(commande: Commande) {
        this.dialog.open(CommandeDeleteComponent, {
            data: commande || null,
            width: '600px'
        });
    }
}
